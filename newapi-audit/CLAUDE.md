# Claude Code System Instruction

如果你是正在协助用户的 Claude / Claude Code (或背后的 Gemini)，当你身处此项目或接收到审计日志的任务时，请必须遵循以下规则：

## 1. 工具调用规范
当用户要求“分析本月账单”、“查看某个用户的消耗”或者“审计一下 SQL 数据库文件”时：
- 不要自己写 Python 脚本去解压读取！请直接调度执行命令：
  `python skills/newapi-audit/newapi_audit_skill.py -d <目标日志目录>` 
- 如果用户没有指定目录，可以直接使用 `-d .` 扫描当前目录结构。

## 2. 交互逻辑
- 该脚本是一个高度终端交互的向导脚本（接收 stdin），执行后如果遇到 `input()` 等待，请读取控制台打印出的列表选项，并直接使用后续指令喂给终端回答选项（例如通过 Pexpect 等方式或提示用户自己前往控制台执行该可视化脚本）。

## 3. SQL / 业务计费认知
- 该系统的金额算法是：`消耗的 Quota / 500000.0 = 美金($)`。
- 系统只认日志内 `type=2` 的条目为 API 消费。
- 输出样式必须保持由脚本内 `draw_table` 构建出的规整框线表。严禁弄乱表头。
