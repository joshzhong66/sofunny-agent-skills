# NewAPI PostgreSQL 流式审计向导工具技能 V8

这是一个专门配合大语言模型 CLI（如 Claude Code、OpenCode 等）及人工交互协同使用的高效 SQL 数据分析与聚合账单导出脚本。旨在通过 `OOM Free (0 内存防爆)` 机制解析数十 MB 甚至高达几 GB 级别的 `NewAPI` / `OneAPI` 数据库备份文件。

### 1. 🚨 项目痛点与背景

在分析由 `pg_dump` 导出的超大 PostgreSQL/MySQL 备份原始文件时，我们通常面临极高的执行门槛：
*   ❌ **文件体积过于庞大：** 解析好几个 GB 的单体 `.gz` 数据块时如果使用 Python 传统 `readlines()` 或使用字符串 `.split()` 会直接撑炸内存导致进程被 OS 强杀 (OOM)。
*   ❌ **压缩格式极其繁杂：** 服务器备份出来的可能是 `*.sql.tar.gz`、`*.gz`、或普通的 `.zip` 环境。人工逐一去下载、解压还原到磁盘往往耗费数十分钟。
*   ❌ **数据序列标准不统一：** 存在部分全量导出使用的是 `COPY public.logs FROM stdin;`，存在其他分发导出使用 `INSERT INTO public.logs (...) VALUES (...);`，这就导致了简单的正则是无法兼容捕获所有流水行的！
*   ❌ **被污染的复杂 String：** NewAPI 的 `content` 字段本身存留了用户请求过的恶心的对话报文，里面可能包含无数个“未转义的半角逗号(,)”，盲目靠 `,` 去 `split` 全部截断错位。

### 2. 🌟 核心突破与重构改进

#### 2.1 架构层优化
| 层面 | 以往解析方案 / 早期 V1-V5版 | 全新 V8 `extract_universal_row` 版 | 重大提升 |
| :--- | :--- | :--- | :--- |
| **内存管控机制** | `f.read()` 全盘压入系统内存 | 原生低级生成器迭代 (流式 `for line` 释放) | **内存缩减 99%** |
| **执行压缩解析** | 人工解压 -> 找文件 -> 粘贴路径执行 | 智能多重调度 (`ZipFile`/`tarfile`/`Gzip`) 落盘 | **0 人工介入** |
| **数据库读取头兼容** | 仅认 `COPY` 导致部分 `.gz` 全漏数据 (0 笔提取) | “混沌状态混合型提取” 完美双轨兼容 `INSERT INTO` | **0 漏抓捕获** |
| **交互与过滤体验** | `input()` 盲填生硬的 `YYYY-MM-DD` | 探测全生命周期的时轴后给出 5 级时区选择抽屉 | **傻瓜化 UI** |
| **前端终幕渲染** | 数据内容由于 UTF-8 特殊编码字符被挤出列外 | 取值 `unicodedata` 动态渲染、双宽截断闭合列格 | **完美等距对齐** |

### 3. 🤖 全平台 AI 技能原生支持 (Agent Native)

本项目除了作为普通的 Python 工具，它更核心的身份是一套支持 AI Terminal （全大模型辅助终端环境，如 Claude Code）的自动化 **Skills 插件包**。
针对项目接的到代码规则约束文档：

1. **`CLAUDE.md`**：让 Claude Code 在该目录下工作时，立刻习得如何解析此工具的交互，并清楚知悉 `Quota(额度) / 50万` 美金计费汇算比例。 
2. **`Codex.md`**：兼容 Github 原生 Copilot 以及 OpenCode IDE 的解析指令。

### 4. 🚀 部署与极速开始

本技能运行时 **无需任何第三方依赖**，完全拥抱 Python 原生生态支持！

```bash
# 进入你想存放克隆包的本地目录
git clone git@github.com:joshzhong66/sofunny-agent-skills.git
cd sofunny-agent-skills

# 执行本命令行工具！(使用 -d / --dir 指定日志的大目录)
python skills/newapi-audit/newapi_audit_skill.py -d "F:\NewAPI\newapi_log_file"

# 如果你现在就处于你的日志文件夹下，直接省略目录参数执行：
python skills/newapi-audit/newapi_audit_skill.py
```
